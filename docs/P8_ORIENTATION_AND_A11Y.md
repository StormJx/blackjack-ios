# P8 横竖屏 / 无障碍（后续）

> 深色模式基础适配已在 `TableBackgroundView` 落地（跟随系统）。  
> 下列两项**未实现**，仅备案，点名后再做。

---

## 横竖屏（后续）

现状：工程仅支持竖屏（`UISupportedInterfaceOrientations = Portrait`）。

建议后续：

1. 评估是否允许横屏；若允许，牌桌操作区与手牌区需重排（横屏高度紧）。
2. iPad / 分屏尺寸变化时避免裁切「确认并发牌」等主按钮。
3. 默认仍可保持竖屏优先；横屏为增强项。

---

## 无障碍深化（后续）

现状：部分控件有 `accessibilityLabel`；发牌尊重减弱动态效果；牌面有读法。

建议后续：

1. VoiceOver：点数、注码、结果弹层、道具按钮完整可读。
2. 动态字体：大字号下按钮与欢迎页不严重截断。
3. 对比度：深色模式下结果色 / 禁用态抽查。
4. 不单靠颜色传达胜负（可保留图标 + 文案）。
